# Horizon Capital — Terraform

Provisions the AWS footprint for the firm: VPC, RDS Postgres, ElastiCache
Redis, ECR, ECS Fargate (web + worker), ALB, IAM, SSM secrets, and
CloudWatch log groups.

## Layout
```
infra/terraform/
├── main.tf            # composition root (all resources, inline)
├── envs/
│   ├── dev.tfvars     # workspace = dev
│   └── prod.tfvars    # workspace = prod
└── README.md
```

## Usage

```bash
cd infra/terraform

# one-time
terraform init

# create / select workspace per env
terraform workspace new dev      # or: terraform workspace select dev

# plan & apply
terraform plan  -var-file=envs/dev.tfvars
terraform apply -var-file=envs/dev.tfvars

# tear down (dev only — prod has deletion protection)
terraform destroy -var-file=envs/dev.tfvars
```

## State backend

For production, uncomment and configure the S3 backend in `main.tf` (with
a DynamoDB lock table). Until then state is local — never commit `*.tfstate*`.

## Secrets

OpenAI keys go to SSM Parameter Store. Pass via:

```bash
terraform apply -var-file=envs/dev.tfvars -var="openai_api_key=$OPENAI_API_KEY"
```

CI/CD should source the key from its own secret store, not env. The DB
password is auto-generated by `random_password` and stored in SSM.

## Cost guard

Dev defaults to single-AZ RDS, 1 web task, 1 worker. Prod uses multi-AZ
RDS, 3 web replicas, 2 workers. Adjust in tfvars per environment.

## Outputs

After apply:

| Output             | Use                                                          |
|--------------------|--------------------------------------------------------------|
| `alb_dns`          | Browse the firm UI                                           |
| `ecr_repository_url` | Push image: `docker tag ... && docker push ...`           |
| `rds_endpoint`     | DB host (sensitive)                                          |
| `redis_endpoint`   | Cache host                                                   |
| `cluster_name`     | `aws ecs update-service --cluster ... --force-new-deployment`|
