1. Scheduler — 3 לולאות רקע + 5 משימות שעון השוק

המתזמן (app/scheduler.py) מריץ 3 לולאות רקע. רק market_cadence_loop מפיץ ל-5 משימות שעון השוק (ET). שתי הלולאות האחרות מריצות פעולה בודדת כל אינטרוול.

2. RAG — מה נכנס, מה יוצא, ומי משתמש

4 קורפוסים נכנסים דרך seed.py ו-rag_bootstrap.ensure_ready(), מקודדים ל-vector store ב-SQLite. rag.search() מוגש לסוכנים דרך 4 כלים ב-tools.py.

3. 9 הסוכנים (בנפרד)

3.1 auditor

LLM-as-judge שבודק פלט של כל סוכן בדיעבד. נקרא דרך graph._run_agent אחרי כל קריאת סוכן.

3.2 firm_manager

Portfolio Manager — policy + routing בלבד; לא קובע trades. נקרא ב-pre_open / market_open / eod.

3.3 fundamental

Fundamental Analyst — בונה thesis מ-data + RAG. Refuses honestly אם data unavailable.

3.4 idea_generator

סורק universe באופן יזום, מסנן novelty, מדרג composite (quality + valuation + precedent + fit) ומפיק שני יציאות: ranked briefs ו-spawn news_runs.

3.5 news_triage

נקודת הכניסה ל-news pipeline. ציון materiality מול הספר ומול ה-policy; מחליט אם המידע ראוי לתכנון.

3.6 plan_builder

בונה תוכנית entry/exit עם sizing על בסיס thesis. Refuses אם quote unavailable; מעביר mechanical gates לפני LLM.

3.7 plan_supervisor

Smart gate לפני execution + stewardship שוטף. מקבל risk_context וגם monitor_report; לא עוקף Firm Charter §4 או HITL.

3.8 position_monitor

בודק holdings פתוחים מול monitoring.checks של ה-plan (price_drift, news, filings, earnings) + guardrails + exit_rules. לעולם לא auto-trades.

3.9 risk_officer

בודק תוכנית מול policy + concentration + correlation. קובע אם נדרש HITL (risk_requires_hitl) ומספק verdict עם reasoning ו-policy citations.

4. דיאגרמת אינטראקציה כוללת — כל הסוכנים

שלוש כניסות (News event + 2 cadence loops) מזינות 3 ענפים שמתאחדים ב-plan_supervisor: news pipeline (מרכז), idea pipeline (שמאל), position monitoring (ימין). auditor רץ cross-cutting אחרי כל סוכן.
